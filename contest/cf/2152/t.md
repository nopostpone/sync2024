# 问题描述

这是一个交互题。

Faker 又在调皮了。你让他出一个好的查询题，但他却出了一个交互题，而他是回答查询的那个人！Faker 向你隐藏了一个排列，你需要通过与他交互来推断一些有趣的信息。

你被给定一个整数 \( n \)。Faker 隐藏了一个长度为 \( n^2 + 1 \) 的隐藏排列 \( p_1, p_2, \ldots, p_{n^2 + 1} \)。你的目标是找出一个长度恰好为 \( n + 1 \) 的单调子序列（递增或递减）。

为了找到它，你可以向交互器提出最多 \( n \) 次 **查询**，定义如下：

- 你提供一个严格递增的索引序列：\( i_1, i_2, \ldots, i_k \)。
- 交互器考虑这些索引在隐藏排列中的值：\( p_{i_1}, p_{i_2}, \ldots, p_{i_k} \)。
- 交互器返回该集合中可见的索引。一个索引 \( i_j \) 是可见的，如果它的值 \( p_{i_j} \) 大于你查询中所有前面元素的值，即对于所有 \( 1 \leq m < j \)，有 \( p_{i_j} > p_{i_m} \)。这等价于找到序列 \( (p_{i_1}, \ldots, p_{i_k}) \) 的从左到右的最大值索引。

在最多进行 \( n \) 次查询后，你必须报告一个长度**恰好为 \( n + 1 \)** 的有效单调子序列。

注意，排列 \( p \) 在**任何查询之前**就已经固定，并且不依赖于查询。

---

# 输入格式

每个测试包含多个测试用例。第一行包含测试用例的数量 \( t \)（\( 1 \leq t \leq 5000 \)）。测试用例的描述如下。

每个测试用例的第一行也是唯一一行包含一个整数 \( n \)（\( 1 \leq n \leq 100 \)）。

保证所有测试用例的 \( n^2 + 1 \) 的总和不超过 10,001。

---

# 交互流程

每个测试用例的交互从读取整数 \( n \) 开始。

要进行查询，请按以下格式打印一行：

```
? k i₁ i₂ … iₖ
```

其中 \( k \) 是查询中的索引数量（\( 1 \leq k \leq n^2 + 1 \)），\( i_1, \ldots, i_k \) 是索引本身，满足 \( 1 \leq i_1 < i_2 < \ldots < i_k \leq n^2 + 1 \)。索引应按升序排列。

作为响应，交互器将按以下格式打印一行：

```
c j₁ j₂ … j_c
```

其中 \( c \) 是查询中可见的摩天大楼数量（\( 1 \leq c \leq k \)），\( j_1, \ldots, j_c \) 是它们的索引，满足 \( 1 \leq j_1 < j_2 < \ldots < j_c \leq n^2 + 1 \)。索引将按升序排列。

要报告最终答案，请按以下格式打印一行：

```
! s₁ s₂ … s_{n+1}
```

其中 \( s_1, \ldots, s_{n+1} \) 是构成你找到的长度为 \( n + 1 \) 的单调子序列的元素的索引，满足 \( 1 \leq s_1 < s_2 < \ldots < s_{n+1} \leq n^2 + 1 \)。索引应按升序排列。

注意：回答不计入你的命令限制。
